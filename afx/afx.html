afx.base module¶

afx.base.deprecated(func)[source]¶

This is a decorator which can be used to mark functions as deprecated. It will result in a warning being emitted when the function is used.

afx.base.exhandler(function, parser)[source]¶

If -examples was specified in ‘args’, the specified function is called and the application exits.

Parameters:
  • function – the function that prints the examples.
  • parser – the initialized instance of the parser that has the additional, script-specific parameters.
afx.base.set_testmode(mode_)[source]¶

Sets the unit testing mode of the package.

afx.base.testmode = False¶

bool – when True, the package is running in unit test mode; this changes the behavior of requests to remote resources to use local emulators instead.

afx.msg module¶

This module handles writing to the terminal or a log file with support for coloring for warnings, errors, etc.

afx.msg.arb(text, cols, split)[source]¶

Prints a line of text in arbitrary colors specified by the numeric values contained in msg.cenum dictionary.

afx.msg.blank(n=1, level=2)[source]¶

Prints a blank line to the terminal.

afx.msg.cenum = {'cwarn': 1, 'cinfo': 2, 'cgens': 3, 'cstds': 4, 'cerrs': 0, 'cokay': 5}¶

Dict of the various colors available for coloring specific lines in the arb().

afx.msg.err(msg, level=-1, prefix=True)[source]¶

Prints the specified message as an error; prepends “ERROR” to the message, so that can be left off.

afx.msg.gen(msg, level=1)[source]¶

Prints the message as generic output to terminal.

afx.msg.info(msg, level=1)[source]¶

Prints the specified message as information.

afx.msg.nocolor = False¶

When true, the colored outputs all use the regular print() instead so that the stdout looks ordinary.

afx.msg.okay(msg, level=1)[source]¶

Prints the specified message as textual progress update.

afx.msg.printer(text, color=None, **kwargs)[source]¶

Prints using color or standard print() depending on the value of ‘nocolor’.

afx.msg.quiet = None¶

When in quiet mode, no text is ever printed to terminal unless its verbosity level is explicitly specified.

afx.msg.set_quiet(is_quiet)[source]¶

Sets whether the messaging system is running in quiet mode. Quiet mode only prints messages with explicit verbosity specified. If verbosity==None and quiet==True, any message with level >= 0 is printed.

afx.msg.set_verbosity(level)[source]¶

Sets the modules message verbosity level for all messages printed.

Parameters:level – a positive integer (>0); higher levels including more detail.
afx.msg.std(msg, level=1)[source]¶

Prints using the standard print() function.

afx.msg.verbosity = None¶

The verbosity level of messages being printed by the module.

afx.msg.warn(msg, level=0, prefix=True)[source]¶

Prints the specified message as a warning; prepends “WARNING” to the message, so that can be left off.

afx.msg.will_print(level=1)[source]¶

Returns True if the current global status of messaging would print a message using any of the printing functions in this module.

afx.preprocessing module¶

Functions for pre-processing data before transformations. If the function you are adding applies a transformation, then put it in the relevant place in ./transforms sub-package. Examples of what belongs here:

  • data munging
  • imputing missing values

afx.utility module¶

Utility functions for I/O, system calls and visualization.

afx.utility.reporoot = '/home/david/super-ai'¶

The absolute path to the repo root on the local machine.
